# Webhook API Reference

The MWP pipeline exposes a REST API via Modal for triggering episode generation and checking job status.

## Base URL

```
https://YOUR_MODAL_USERNAME--mwp-recording-app-web.modal.run
```

## Endpoints

### GET /

Returns API info and available endpoints.

**Response:**
```json
{
  "name": "MWP Episode Generator API",
  "endpoints": { ... }
}
```

### GET /health

Health check endpoint.

**Response:**
```json
{
  "status": "ok"
}
```

### GET /docs

Swagger/OpenAPI documentation (auto-generated by FastAPI).

### GET /jobs

List running and recently completed jobs.

**Response:**
```json
{
  "running": [...],
  "recent": [...]
}
```

### GET /status/{job_id}

Check the progress of a generation job.

**Response:**
```json
{
  "job_id": "abc123",
  "status": "running",
  "stage": "Generating TTS audio...",
  "progress": 65
}
```

### POST /webhook/test

Test endpoint that validates the payload format without triggering generation. No authentication required.

**Request:**
```json
{
  "audio_url": "https://example.com/test.mp3"
}
```

**Response:**
```json
{
  "status": "ok",
  "message": "Payload validated",
  "audio_url": "https://example.com/test.mp3"
}
```

### POST /webhook/generate

Generate a full episode from an audio URL. Requires authentication.

**Headers:**
| Header | Required | Description |
|--------|----------|-------------|
| `Content-Type` | Yes | `application/json` |
| `X-Webhook-Secret` | Yes | Must match the `WEBHOOK_SECRET` environment variable |

**Request Body:**

```json
{
  "audio_url": "https://example.com/recording.mp3",
  "sender": {
    "type": "daniel"
  },
  "attachment_url": "https://example.com/context.pdf"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `audio_url` | Yes | URL to the audio recording (MP3, WAV, etc.) |
| `sender` | No | Sender information (see Sender Types below) |
| `sender.type` | No | `daniel` (default), `hannah`, or `other` |
| `sender.name` | No | Required if `type` is `other` |
| `sender.description` | No | Optional description for `other` sender type |
| `attachment_url` | No | URL to supplementary context file |

**Fillout Form Format:**

The endpoint also accepts Fillout form submissions:
```json
{
  "responses": [
    {
      "type": "voice_recording",
      "value": "https://cdn.fillout.com/recording.mp3"
    }
  ]
}
```

**Response:**
```json
{
  "status": "accepted",
  "job_id": "abc123-def456",
  "message": "Episode generation started"
}
```

**Error Responses:**

| Status | Description |
|--------|-------------|
| 401 | Invalid or missing `X-Webhook-Secret` |
| 400 | Invalid payload (missing `audio_url`) |
| 500 | Internal server error |

### POST /webhook/generate-from-script

Generate an episode from a pre-written diarized script, skipping transcription, research, planning, and script generation. Requires authentication.

**Headers:** Same as `/webhook/generate`.

**Request Body:**
```json
{
  "script": "CORN: Welcome to the show!\nHERMAN: Thanks Corn, today we're diving into..."
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `script` | Yes | Pre-written diarized script (speaker labels required) |

The script must use the format `SPEAKER: dialogue text` with one segment per line.

## Sender Types

| Type | Description |
|------|-------------|
| `daniel` | Default — Daniel, the housemate who sends prompts |
| `hannah` | Hannah (Daniel's wife) — introduced as a special guest episode |
| `other` | External guest — requires `name` field, optional `description` |

## Authentication

The `/webhook/generate` and `/webhook/generate-from-script` endpoints require the `X-Webhook-Secret` header to match the `WEBHOOK_SECRET` environment variable configured in Modal secrets.

```bash
curl -X POST "https://YOUR_URL/webhook/generate" \
  -H "Content-Type: application/json" \
  -H "X-Webhook-Secret: your-secret-here" \
  -d '{"audio_url": "https://example.com/recording.mp3"}'
```

## Publication Webhook (Outbound)

After an episode is published, the pipeline sends a POST request to the configured publication webhook URL with full episode data. This enables downstream syndication to platforms like Telegram, Twitter/X, Substack, etc.

See `pipeline/webhooks/publication_webhook.py` for the full payload structure. Configure via `PUBLICATION_WEBHOOK_URL_PROD` and `PUBLICATION_WEBHOOK_URL_TEST` environment variables.
