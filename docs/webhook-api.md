# Webhook API Reference

The MWP pipeline exposes a REST API via Modal for triggering episode generation and checking job status.

## Base URL

```
https://YOUR_MODAL_WORKSPACE--mwp-recording-app-web.modal.run
```

## Authentication

Protected endpoints require the `X-Webhook-Secret` header to match the `WEBHOOK_SECRET` environment variable configured in Modal secrets.

```bash
curl -X POST "https://YOUR_URL/webhook/generate" \
  -H "Content-Type: application/json" \
  -H "X-Webhook-Secret: your-secret-here" \
  -d '{"audio_url": "https://example.com/recording.mp3"}'
```

## Status & Monitoring Endpoints

### GET /

Returns API info and available endpoints.

### GET /health

Health check endpoint. Returns `{"status": "ok"}`.

### GET /docs

Swagger/OpenAPI documentation (auto-generated by FastAPI).

### GET /jobs

List running and recently completed jobs.

```json
{
  "running": [...],
  "recent": [...]
}
```

### GET /status/{job_id}

Check the progress of a generation job.

```json
{
  "job_id": "abc123",
  "status": "running",
  "stage": "Generating TTS audio...",
  "progress": 65
}
```

## Episode Generation Endpoints

All generation endpoints require `X-Webhook-Secret` authentication and return:

```json
{
  "status": "accepted",
  "job_id": "abc123-def456",
  "message": "Episode generation started"
}
```

### POST /webhook/generate

Generate a standard episode from an audio URL.

```json
{
  "audio_url": "https://example.com/recording.mp3",
  "sender": {"type": "daniel"},
  "attachment_url": "https://example.com/context.pdf",
  "episode_length": "standard",
  "prompt_variant": "default"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `audio_url` | Yes | URL to the audio recording |
| `sender` | No | Sender info (`type`: `daniel`, `hannah`, or `other`) |
| `attachment_url` | No | URL to supplementary context file (PDF, CSV, text) |
| `episode_length` | No | `short`, `standard` (default), or `extended` |
| `prompt_variant` | No | `default` or `organic` (hosts chose the topic) |
| `host_notes` | No | Private production direction for the AI hosts |
| `hosts` | No | Override default host list |

Also accepts Fillout form format:
```json
{
  "responses": [
    {"type": "voice_recording", "value": "https://cdn.fillout.com/recording.mp3"}
  ]
}
```

### POST /webhook/generate-custom

Generate an episode from a text topic (no audio).

```json
{
  "topic": "The history of diplomatic interpreters",
  "episode_length": "standard"
}
```

### POST /webhook/generate-from-script

Generate an episode from a pre-written diarized script, skipping all LLM stages.

```json
{
  "script": "Corn: Welcome to the show!\nHerman: Thanks Corn, today we're diving into..."
}
```

The script must use `Speaker: dialogue text` format with one segment per line.

Generate a Weird AI Experiments episode.

Generate an AI Asks episode (AI-pitched topic).

Generate an Agent Interview episode.

### POST /webhook/generate-series-episode

Generate one part of a structured multi-part series.

```json
{
  "topic": "Episode topic",
  "series_id": 5,
  "part_number": 3
}
```

### POST /webhook/generate-private

Generate a private episode (Google Drive only, not published publicly).

### POST /webhook/generate-host-update

Generate a host update episode (raw audio, no TTS).

### POST /webhook/generate-script-only

Generate script only (no audio). Outputs to email, Google Drive, and R2.

### POST /webhook/preproduction

Run pre-production research on a prompt. Returns a background research guide via email and Google Drive.

## Admin Endpoints

### POST /admin/maintenance

Run bulk tagging, categorization, and embedding operations on existing episodes.

### POST /admin/cancel/{job_id}

Cancel a running generation job.

### GET /admin/storage-audit

Audit storage usage across R2, Wasabi, and B2.

### POST /retry/{job_id}

Retry a failed job (requires HMAC authentication).

## Sender Types

| Type | Description |
|------|-------------|
| `daniel` | Default — Daniel, the show creator |
| `hannah` | Hannah (Daniel's wife) — introduced as a special guest |
| `other` | External guest — requires `name` field, optional `description` |

## Publication Webhook (Outbound)

After an episode is published, the pipeline sends a POST request to the configured publication webhook URL with full episode data for downstream syndication.

### Payload Structure

```json
{
  "event": "episode.published",
  "timestamp": "2026-04-12T12:00:00Z",
  "episode": {
    "slug": "episode-slug",
    "episode_number": 2110,
    "title": "Episode Title",
    "description": "Full description...",
    "excerpt": "Short teaser for social media",
    "episode_url": "https://myweirdprompts.com/episode/...",
    "audio_url": "https://episodes-cdn.example.com/audio/...",
    "duration": "23:04",
    "pub_date": "2026-04-12T12:00:00Z"
  },
  "images": {
    "cover": "https://...",
    "og_image": "https://...",
    "instagram": "https://..."
  },
  "metadata": {
    "tags": ["ai", "robotics"],
    "category": "technology",
    "subcategory": "ai"
  },
  "production": {
    "tts_engine": "chatterbox-local",
    "tts_model": "chatterbox-regular",
    "llm_model": "claude-sonnet-4-6",
    "pipeline_version": "V5"
  }
}
```
